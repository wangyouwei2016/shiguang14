# 觉行三境（独语/灵思/问程）设计

## 目标

将原「灵感池」升级为「觉行三境」，并在其中按三个分区呈现与管理内容：

- 独语：自己与自己的对话（从灵思可切换而来，也可切回）
- 灵思：灵感任务/闪念（默认入口）
- 问程：原目标管理（仅改名与文案，不改核心交互）

## 数据模型

在 `Task` 上新增可选字段 `ideaRealm` 用于对 `status === 'idea'` 的任务分区：

- `ideaRealm: 'duyu' | 'lingsi'`
- 兼容旧数据：旧任务没有 `ideaRealm` 时，视为 `lingsi`（灵思）
- 新建 idea 任务默认 `ideaRealm = 'lingsi'`

该字段不用于非 idea 状态（`focus/today/completed`），以避免扩散语义。

## UI/交互

### 导航与页面标题

- 侧边栏入口文案从「灵感池」改为「觉行三境」
- 页面顶部标题改为「觉行三境」，并保持现有整体视觉风格

### 分区与顺序

「觉行三境」页面内分区按顺序展示：

1. 独语（标题说明：内观·明心见性）
2. 灵思（标题说明：入微·步步为营）
3. 问程（标题说明：立志·知行合一）

其中「独语」「灵思」的分区标题字体与图标风格保持与「问程」一致（同字号/字重/字距、左侧图标同尺寸与颜色）。

### 任务切换

仅在 idea 任务上提供 realm 切换：

- 灵思条目提供「移到独语」按钮：`ideaRealm = 'duyu'`
- 独语条目提供「移到灵思」按钮：`ideaRealm = 'lingsi'`

切换通过现有 `updateTask(id, patch)` 持久化到 `data/tasks.json`。

## API 与校验

`/api/tasks` 的任务 schema 校验增加可选字段 `ideaRealm`：

- 缺省允许
- 若存在，必须为 `'duyu' | 'lingsi'`

## 风险与约束

- 由于 UI 分区可能引入重复渲染逻辑，需要抽出复用组件以避免单文件超过 300 行、单函数超过 50 行等硬性指标。

## 验证方式

- `npm run build` 必须通过（TypeScript/Next 编译通过）
- 手工验证：
  - 新增闪念默认进入「灵思」
  - 灵思条目可切到独语，独语条目可切回灵思
  - 编辑/删除/移入行囊等按钮功能不回归
