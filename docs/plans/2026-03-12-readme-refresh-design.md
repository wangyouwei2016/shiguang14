# README 改造设计（中文优先，3 分钟上手）

## 背景
当前 README 信息完整，但内容偏长，首次访问者在“快速启动、登录配置、数据路径”上的关键信息需要下钻后才能看到，导致上手路径不够短。

## 目标
将 README 重排为“先上手、后深入”的结构，在不丢失关键技术信息的前提下，让新用户可在 3-5 分钟内完成本地启动与登录。

## 设计原则
1. 中文优先：主体内容中文表达，英文仅保留最小摘要。
2. 可执行优先：命令、路径、变量与接口优先于背景叙述。
3. 单点讲透：同一主题只在一个章节完整解释，避免重复。
4. 兼容现状：所有说明以当前代码实现为准，不引入假设能力。

## 信息架构
1. 项目标题与一句话定位
2. 30 秒看懂（核心闭环、适用场景、原型状态）
3. 3 分钟启动（环境、安装、配置、启动、登录）
4. 核心功能（压缩版）
5. 配置说明（必须与可选变量）
6. 数据与存储（单用户/多用户路径与边界）
7. API 概览
8. Docker 部署
9. 常见问题（FAQ）
10. 详细文档索引（docs）

## 内容迁移策略
- 前移：启动命令、登录二选一规则、数据路径、Docker 挂载。
- 合并：亮点/架构/风格说明，减少重复叙述。
- 下沉：历史背景、未来演进到文末简述或删除。
- 替换：将 GEMINI 相关变量标注为“可选/模板遗留”。

## 验收标准
1. 从 README 单文档可完成本地启动与登录。
2. 登录配置二选一规则明确且示例可直接复制。
3. 数据文件路径对单用户与多用户模式都清晰可见。
4. Docker 命令可直接运行并包含持久化挂载。
5. API、脚本与 docs 索引仍可快速定位。
6. 文档风格中文优先，无整段中英重复。
